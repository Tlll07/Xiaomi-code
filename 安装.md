## 第一部分：认识 MiMo Code

MiMo Code 是小米技术团队推出的一款**终端原生的 AI 编程助手**，作为个人开发者，你完全免费开源使用。

*   **自主进化 (Self-Evolve)**：内置 `/dream` 智能体，定期扫描历史对话，将学到的关于你项目的知识、模式更新到记忆文件中，实现“越用越懂你”。
*   **多 Provider 支持**：内置了对 75+ 家 AI 服务商的支持，这意味着你既可使用 MiMo 的免费模型，也能接入 DeepSeek、OpenAI 等自己订阅的 API。(会在下面教你如何操作)
*   **开源免费**：项目基于 MIT 协议开源，可以免费用于个人和企业。
*   **零配置上手 (MiMo Auto)**：内置的限时免费模型 `MiMo Auto` (即 `MiMo-V2.5`)，开箱即用，无需注册和 API Key。(免费是有限制的会在最后讲)
*   **持久化记忆 (Persistent Memory)**：告别“一关就忘”。它能跨会话记住项目的架构、决策和你的偏好。
*   **多智能体协作 (Multi-Agent)**：通过三种工作模式调用专业智能体，并按需生成并行子智能体高效处理任务。


## 第二部分：安装指南 (Windows 11)

### 2.1 环境准备
1.  **安装 Node.js**：MiMo Code 依赖 Node.js 环境。
    *   **操作**：从 [Node.js 官网](https://nodejs.org) 下载并安装，推荐 **v20 LTS** 版本。
    *   **验证**：安装成功后，打开命令提示符（cmd）或 PowerShell，输入 `node -v` 和 `npm -v`，如果显示版本号，则说明安装成功。
2.  **Windows Terminal (强烈推荐)**：从微软商店下载安装，它能提供比原生 cmd 更好的终端体验。(话是如此但win11原生个人用起来还可以)

### 2.2 安装步骤
1.  **配置 npm 国内镜像源 (加速)**：
    ```bash
    npm config set registry https://registry.npmmirror.com
    ```
2.  **全局安装 MiMo Code**：
    ```bash
    npm install -g @mimo-ai/cli
    ```
    *   **注意**：如果过程中出现卡顿，请耐心等待，安装过程可能需要 1-2 分钟。
3.  **验证安装**：
    ```bash
    mimo --version
    ```
    如果显示版本号如 `0.1.0`，则说明安装成功。


## 第三部分：初始配置与启动

### 3.1 首次启动与选择模型
1.  **启动**：在你的项目目录中打开终端，输入 `mimo` 并回车。
2.  **选择模型**：启动时会引导你选择 AI 模型。
    *   **推荐选择**：`MiMo Auto (MiMo-V2.5 限免中)`。这是小米官方提供的免费模型，**无需任何 API Key，开箱即用**。
    *   **其他选项**：如需接入 DeepSeek 或 OpenAI 等其他模型，可按提示配置 API Key。

### 3.2 模型配置 (可选)
如果你需要接入自己的模型，有以下两种方式：

*   **交互式配置**：在 MiMo Code 启动后的聊天界面输入命令 `/connect`，然后根据终端提示选择提供商并粘贴你的 API Key。
*   **命令行配置**：直接在终端中用命令添加 API Key。
    ```bash
    # 以接入 DeepSeek 为例
    echo sk-你的DeepSeek_API密钥 | mimo providers login --provider deepseek --method apikey
    # 以接入小米 MiMo API 为例
    echo sk-你的小米API密钥 | mimo providers login --provider xiaomi --method apikey
    ```

### 3.3 故障排查：解决 401 错误
这个通常是因为配置文件中留下了错误的 API Key 信息。

*   **解决方法**：删除配置文件可以强制 MiMo Code 重置所有设置。
    ```bash
    del /s /q %USERPROFILE%\.mimo\*.json 2>nul
    del /s /q %USERPROFILE%\.config\mimocode\*.json 2>nul
    ```
*   **操作后**：重新运行 `mimo`，并确保选择 `MiMo Auto (MiMo-V2.5 限免中)`，即可正常使用。


## 第四部分：核心使用指南

### 4.1 基础操作
*   **开启新会话**：每次进入项目目录，输入 `mimo` 即可开始新会话。
*   **初始化项目**：首次进入项目时，运行 `/init` 为项目建立记忆数据库。
*   **退出**：输入 `exit` 或按 `Ctrl+C` 即可。

### 4.2 工作模式 (核心特色)
这是 MiMo Code 区别于普通 AI 编程工具的核心所在。你可以通过按键盘上的 **`Tab` 键**来快速切换模式。

*   **Plan 模式（只读分析）**
    *   **角色**：项目的架构师与分析师。
    *   **权限**：**只读**，无法修改文件。
    *   **最佳场景**：在动手写代码前，先用它探索代码库、分析架构、提出技术方案，或进行代码审查。

*   **Build 模式（执行开发）**
    *   **角色**：高效的执行工程师。
    *   **权限**：**完全访问**。
    *   **最佳场景**：当你已有明确的编程任务时，切换到 Build 模式，它会直接动手写代码。

*   **Compose 模式（规格驱动开发）**
    *   **角色**：专业的开发团队指挥。
    *   **权限**：**完全访问**。
    *   **最佳场景**：应对需要多个步骤的复杂开发任务，输入一个想法，它就能自动完成设计、规划、编码、测试和审查的完整流程。

### 4.3 核心命令
在 MiMo Code 的聊天界面，输入 `/` 可以唤起命令列表：

*   `/connect`：配置或切换 LLM 服务商（OpenAI, DeepSeek, MiMo...）。
*   `/clear`：清空当前会话上下文。
*   `/model`：查看或切换当前使用的模型。
*   `/help`：查看所有支持的命令。
*   `/voice`：启动语音输入功能，MiMo Code 会实时将你的语音转成文字指令。
*   `/add`：将文件或目录添加到当前会话上下文中。
*   `/goal`：设定一个任务完成目标，AI 会持续工作直到满足你设定的条件。
*   `/dream`：手动触发“梦境”，让 AI 回顾和整理历史会话，提取有价值的信息并将其转化为长期记忆。
*   `/distill`：梳理历史，总结出重复出现的工作模式，并将其打包成一个可复用的“技能 (Skill)”。


## 第五部分：使用技巧与最佳实践

**1. 高效的工作流：Plan -> Build**
这是一个非常高效且值得养成的习惯：先用 **Plan 模式**制定方案，确认无误后，按 `Tab` 键切换到 **Build 模式**，让它直接执行方案。

**2. Compose 模式的使用**
*   **场景**：当面对一个需要十几步甚至更多步骤才能完成的复杂任务时。
*   **优势**：前期规划的时间更长，但因为一次性把事情做对，最终产出的代码质量更高、更完整，项目总耗时反而可能更短。
*   **建议**：可以从 `Compose` 开始，它会帮你把复杂任务拆解成规划、编码、测试等多个阶段。

**3. Goal（目标）机制**
*   **用法**：当把 AI 放到后台让它自主工作时，在指令末尾补充 `/goal [具体标准]`，可以让它多一道“检查程序”。
*   **对比**：不加 Goal 可能会在一个子任务完成后就停止；加上 Goal 会促使它对照最终目标验证是否真正完成。

**4. 项目上下文管理**
*   **善用 /add**：开始对话时，用 `/add` 命令添加核心文件，可以确保 AI 始终基于最新的代码进行思考。
*   **查看状态看板**：右侧常驻状态看板会显示任务树 (Task Tree)，可以帮你直观地跟踪进度。

## ❓ 遇到问题？快速排查手册

| 你可能遇到的问题 | 原因 | 解决方法 |
| :--- | :--- | :--- |
| `npm install` 命令卡住不动 | 网络连接 `npm` 官方源缓慢 | 配置国内镜像源：`npm config set registry https://registry.npmmirror.com`，然后重新安装。 |
| `mimo` 命令提示“不是内部或外部命令” | 系统环境变量 `PATH` 未包含 npm 全局安装目录 | 手动将 `C:\Users\你的用户名\AppData\Roaming\npm` 添加到系统环境变量 `Path` 中。 |
| 启动后出现 `✗ Invalid API Key` 错误 (401) | 配置文件内有错误的 API Key 信息 | 删除配置文件 (`%USERPROFILE%\.mimo\` 或 `%USERPROFILE%\.config\mimocode\`)，然后重新启动 `mimo`，选择 `MiMo Auto`。 |
| 如何升级 MiMo Code 到最新版本？ | - | 运行 `npm update -g @mimo-ai/cli` 命令。 |
| 对话上下文快满了，但任务还没完成怎么办？ | 超过了模型处理的上下文长度限制 | MiMo Code 会自动触发“检查点”机制。它会保存当前状态，清理上下文，然后从保存点继续，无需你手动干预。 |
